- (idea) Multi-LLM support (not just Ollama)
- (idea) Python implementation track alongside Go
- (idea) Build a terminal UI like Claude Code for the agent
- (idea) Build a full CLI like Claude Code (the whole CLI experience, not just one moment)
- (idea) Takeaway: attendees fully understand how an agent works (demystify the magic)
- (technique) started Devil's Advocate Courtroom
- (insight) Prosecution: attendee may already have built an agent with LangChain/framework — what does from-scratch add?
- (insight) Prosecution: attendees don't NEED to build from scratch in real life — frameworks exist; this is understanding-only, not production
- (idea) Defense: frameworks evolve fast + carry security/supply-chain risk — understanding internals lets you audit, debug, and not blindly trust them
- (idea) Defense closing: after building from scratch, attendee reads any framework's source and sees through it — full demystification, no more black box
- (insight) Jury verdict: workshop's core promise is DEMYSTIFICATION — read any agent code afterward and understand it
- (idea) Ending payoff: attendees launch their own compiled binary and use it like the Claude Code CLI — their own working agent
- (idea) Trait to steal from Claude Code: simple surface, deep ceiling (easy to start, endless depth) — graft onto workshop agent
- (idea) Minute-one hook: attendee gives agent a coding task and watches it actually change files as asked — instant 'I made an agent' feeling
- (idea) Deep ceiling: designing a better agent — architecture decisions, planning loops, memory, multi-step reasoning
- (direction) Hard constraint: 1 hour total for the entire workshop
- (idea) Cut: advanced Go concepts, fancy abstractions — keep vanilla Go only, no external deps beyond Ollama client
- (insight) Spine: vanilla Go + 1h + 'not for prod' are the strength — constraints make demystification visible, nothing hidden
- (insight) Minute-one hook = final takeaway: watch it change a file at start, understand exactly why by the end
- (insight) Workshop IS the agent design: simple surface (understand the loop) → deep ceiling (redesign it) — one path, two depths, no explicit fork
