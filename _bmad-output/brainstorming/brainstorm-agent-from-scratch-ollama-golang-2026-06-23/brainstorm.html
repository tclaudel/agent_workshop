devil's advocate courtroom

Why Build From Scratch at All?

⚖️ PROCEEDINGS: THE CASE FOR FROM-SCRATCH
Prosecution
Attendees may have already shipped agents with LangChain — what's the delta here?
Nobody builds production agents without frameworks. This is understanding-only work.
60 minutes is not enough to learn Go and agents. You're trying to do two things.
Defense
Frameworks evolve fast and carry supply-chain risk. Understanding internals lets you audit, debug, and not blindly trust them.
"Understanding-only" is the point — demystification is the product. Production comes later, informed.
Vanilla Go is the pedagogical tool. Constraints make the mechanism visible. Nothing hidden in a library.
🏛️
Jury Verdict: The workshop's core promise is demystification. After this hour, attendees read any agent codebase — LangChain, CrewAI, AutoGPT — and see through it. No more black box. That's the only outcome that needs to be true.

minute-one magic

The Same Moment, Twice

The workshop opens and closes with the same experience — but the second time, you understand every line of it.

Minute 1 — the hook
Give the agent a task. Watch it change a file.
You type ./agent "refactor this function".

The terminal scrolls. A file gets rewritten. You didn't write the code. You didn't touch the editor.

You have no idea how it happened. That feeling is the entry point.
Minute 60 — the payoff
Run the binary you just built. Watch it change a file.
Same command. Same output. But now you can trace every token: the for loop, the callOllama(), the write_file tool dispatch.

The magic didn't disappear. It became code you own.

workshop arc

60 Minutes, One Thread

0 – 5 min
The Demo
Run a pre-built binary. Give it a task. Watch files change. Plant the question: how is this possible?
5 – 25 min
The LLM Call
Wire up Ollama in vanilla Go. Send a message, receive a response. Learn the chat format. No SDK magic — just HTTP and JSON. First working piece: a function that talks to the model.
25 – 45 min
The Tool Layer
Define tools the model can call: read_file, write_file, run_command. Build the dispatch: parse the LLM's tool-call intent, execute the Go function, feed the result back. Second working piece: the agent can act on the world.
45 – 60 min
The Loop + Ship It
Connect call → tool → observe → repeat into the agentic loop. Compile. Run your own binary on the same task from minute 0. Deep ceiling teaser: how would you add memory? Planning? Multi-step reasoning?

invariants

The Three Constraints That Make It Work

⏱️
Hard Limit
1 hour
Forces ruthless scope. Every feature that doesn't serve demystification gets cut. The constraint is the curriculum.
🔩
No Abstractions
vanilla Go
No LangChain, no SDKs, no agent frameworks. Just stdlib and the Ollama HTTP API. Nothing to hide behind.
🚫
Explicit Non-Goal
not for prod
Saying this out loud frees everyone. You're not learning to ship. You're learning to see. Different standard, different outcome.

synthesis

What the Session Converged On

"One hour. Vanilla Go. Your own binary. The same file-change you saw at minute zero — but this time, you wrote every line that made it happen."

Core Promise
Demystification. Not production readiness. After this workshop, attendees open any agent codebase — LangChain, CrewAI, Claude Code — and recognize the loop, the tool dispatch, the context window management. The magic becomes engineering.
The Teaching Strategy
Constraints as pedagogy. No frameworks means no hiding. Vanilla Go means the mechanism is always visible. "Not for prod" means no premature optimization anxiety. Everything that remains is load-bearing.
The Structural Trick
Same experience, twice. Open with magic, close with authorship. The hook at minute 0 is a question. The payoff at minute 60 is the attendee answering it with a running binary they compiled themselves.
The Deep Ceiling
The workshop ends by opening doors, not closing them. Memory, planning loops, multi-step reasoning — all teased but not taught. The message: you now have the ground truth. Everything else is architecture on top of what you understand.
